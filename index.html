What is CI/CD?

CI/CD is a development practice used in modern cloud systems to automate the process of building and deploying applications. Continuous Integration (CI) means that developers frequently push code changes to a shared repository, where automated checks run to ensure everything works correctly. Continuous Deployment (CD) automatically publishes the application after these checks pass. This approach makes development faster, reduces errors, and allows teams to deliver updates more efficiently.

My Pipeline

In this project, when I push my code to the main branch on GitHub, GitHub Actions automatically starts a workflow. The workflow checks that all required files exist and then prepares the website files. After that, the site is automatically deployed to GitHub Pages so the updated version of the website becomes available online.
